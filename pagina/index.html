Projeto de Monitoramento de Site

Monitoramento de disponibilidade de serviços utilizando AWS e Nginx

Objetivo

O objetivo deste projeto é criar uma solução para monitorar a disponibilidade de um site, enviando notificações em caso de falhas. O monitoramento é realizado periodicamente e, caso o serviço não esteja funcionando, uma notificação é enviada via Discord.

Tecnologias Utilizadas

As tecnologias envolvidas neste projeto incluem:

  • AWS EC2 - Para hospedar o servidor e rodar o script de monitoramento.
  • Nginx - Para servir a página web e monitorar as requisições.
  • Shell Script - Para automatizar a verificação da disponibilidade do site e envio de notificações.
  • Discord Webhook - Para enviar as notificações em caso de falha no serviço.

Funcionamento

O script é executado a cada 1 minuto, verificando se o site está acessível. Se o site estiver fora do ar, o script envia uma mensagem para um canal do Discord, informando a indisponibilidade. O script também armazena logs de suas execuções em um arquivo no servidor.
